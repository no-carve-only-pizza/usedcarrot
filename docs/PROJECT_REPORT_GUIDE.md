# UsedCarrot 최종 보고서 작성 가이드

이 문서는 최종 PDF 보고서를 작성하기 위한 목차와 작성 기준이다. 실제 개발이 진행되면 각 항목에 구현 화면, 코드 일부, 테스트 결과, GitHub 링크를 채워 넣는다.

## 1. 과제 요약

### 1.1 프로젝트명

- UsedCarrot
- Tiny Second-hand Shopping Platform

### 1.2 프로젝트 목적

중고거래 플랫폼의 최소 요구사항을 만족하는 웹 애플리케이션을 개발한다. 개발 전 과정에서 요구사항 분석, 시스템 설계, 구현, 체크리스트 작성, 테스트, 유지보수 계획을 수행하고, 보안 약점이 발견되면 수정 내용을 기록한다.

### 1.3 제출 정보

- 제출물: PDF 보고서, GitHub public repository 링크
- 보고서 포함 사항: 개발 전 과정 설명, 보안 체크리스트, 테스트 결과, 보안 약점 수정 이력, README 실행 방법
- 파일명 형식: `[WHS][secure-coding][XX반]이름(전화번호뒷자리).pdf`
- 주의: 파일명에 공백을 넣지 않는다.

## 2. 요구사항 분석

### 2.1 플랫폼 최소 요구사항

| 요구사항 | 구현 기능 | 관련 문서 |
| --- | --- | --- |
| 사람들이 플랫폼에 가입할 수 있어야 함 | 회원가입, 로그인, 마이페이지 | SRS, AI_IMPLEMENTATION_SPEC |
| 상품들을 올리고 볼 수 있어야 함 | 상품 등록, 상품 목록, 상품 상세, 상품 수정/삭제 | SRS, SystemDesign |
| 플랫폼 사용자끼리 소통 가능 | 상품별 1:1 채팅 | SystemDesign |
| 악성 유저나 상품 차단 | 신고, 상품 숨김, 사용자 제한/정지 | SECURE_CODING_CHECKLIST |
| 유저 간 송금 가능 | CarrotCoin 모의지갑 기반 상품 구매 | AI_IMPLEMENTATION_SPEC |

### 2.2 요구사항 변경 또는 범위 조정

보고서에는 실제 개발 중 변경된 범위를 기록한다.

예시:

- 실시간 전체 채팅은 악용 가능성과 구현 범위를 고려하여 제외하고, 상품별 1:1 채팅을 우선 구현하였다.
- 실제 가상자산 네트워크와 외부 결제 연동은 법적/보안 부담이 크므로 DB 기반 CarrotCoin 모의지갑으로 대체하였다.
- 관심 상품은 최소 요구사항은 아니지만 사용자 편의를 위해 추가 구현하였다.

## 5. 시큐어코딩 적용 내용

### 5.1 인증 보안

- BCrypt 비밀번호 저장
- 로그인 실패 메시지 일반화
- 정지/탈퇴 계정 로그인 차단
- 세션 무효화

### 5.2 인가 보안

- 타인 상품 수정/삭제 차단
- 채팅방 참여자 검증
- 지갑 소유자와 CarrotCoin 거래 참여자 검증
- 관리자 URL 접근 제한

### 5.3 입력값 검증

- DTO Validation
- 서버 측 필수값/길이/형식 검증
- enum 값 제한

### 5.4 XSS 방어

- Thymeleaf `th:text` 사용
- `th:utext` 사용 금지
- 상품명, 설명, 메시지, 신고 내용 escaping

### 5.5 SQL Injection 방어

- Spring Data JPA 사용
- 파라미터 바인딩 사용
- 문자열 연결 SQL 금지

### 5.6 파일 업로드 보안

- 확장자 검증
- MIME 타입 검증
- 파일 크기 제한
- UUID 파일명 저장
- 실행 경로 분리

### 5.7 로그 보안

- 비밀번호, 세션 ID, 토큰, 실제 가상자산 개인키, 외부 API 키 로그 제외
- 로그인 실패, 권한 오류, 신고, 관리자 조치 감사 로그 기록

### 6.2 실제 테스트

테스트 결과는 표로 작성한다.

| 테스트 ID | 테스트 내용 | 결과 | 비고 |
| --- | --- | --- | --- |
| TC-AUTH-001 | 정상 회원가입 | 성공/실패 | 화면 캡처 또는 코드 링크 |
| TC-PROD-003 | 타인 상품 수정 차단 | 성공/실패 | 403 확인 |
| TC-SEC-001 | SQL Injection 검색 방어 | 성공/실패 | 오류 또는 우회 없음 |
| TC-SEC-003 | CSRF 토큰 없는 POST 차단 | 성공/실패 | 요청 거부 확인 |

- 발견 방법
- 수정 전 문제
- 수정 방법
- 수정 후 확인 방법

예시:

| 발견 항목 | 위험도 | 수정 내용 | 확인 방법 |
| --- | --- | --- | --- |
| 상품 수정 시 소유자 검증 누락 | 높음 | ProductService에서 sellerId와 로그인 사용자 ID 비교 추가 | 타인 상품 수정 요청 시 403 확인 |
| 상품 설명 XSS 가능 | 높음 | Thymeleaf th:text 사용, th:utext 제거 | script 입력 후 실행 안 됨 확인 |
| 업로드 파일 확장자만 검증 | 중간 | MIME 타입과 파일 크기 검증 추가 | exe 파일 업로드 실패 확인 |
