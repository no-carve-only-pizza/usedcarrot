# UsedCarrot 시큐어코딩 구현 체크리스트

이 문서는 구현이 끝난 뒤 기능과 보안 요소가 제대로 반영되었는지 점검하기 위한 체크리스트이다. AI에게 구현을 맡긴 뒤, 각 항목을 코드와 실제 실행 화면으로 확인한다.

## 1. 데이터베이스 설계 확인

## 2. 회원가입 및 로그인 보안 체크리스트

| --- | --- | --- | --- |
| 회원가입 | 서버 측 입력값 검증 | 이메일, 비밀번호, 닉네임, 지역에 대해 길이, 형식, 필수값 검증이 서버에서 수행되는지 확인 | [ ] |
| 회원가입 | 이메일 중복 검증 | 이미 가입된 이메일로 회원가입할 수 없는지 확인 | [ ] |
| 회원가입 | 닉네임 중복 검증 | 이미 사용 중인 닉네임으로 회원가입할 수 없는지 확인 | [ ] |
| 회원가입 | 비밀번호 확인 검증 | 비밀번호와 비밀번호 확인이 다르면 가입이 거부되는지 확인 | [ ] |

| 로그인 | 정지 계정 차단 | SUSPENDED 또는 DELETED 상태 사용자가 로그인할 수 없는지 확인 | [ ] |
| 로그인 | 실패 횟수 관리 | 로그인 실패 횟수가 기록되고 기준 이상 실패 시 제한되는지 확인 | [ ] |
| 세션 | 세션 재발급 | 로그인 성공 시 세션 고정 공격 방지를 위해 세션이 재발급되는지 확인 | [ ] |
| 로그아웃 | 세션 무효화 | 로그아웃 후 인증이 필요한 페이지에 접근할 수 없는지 확인 | [ ] |
| CSRF | CSRF 보호 | 회원가입, 로그인, 프로필 수정, 상품 등록 등 상태 변경 요청에 CSRF 보호가 적용되는지 확인 | [ ] |
| 오류 처리 | 내부 정보 숨김 | 오류 응답에 스택 트레이스, DB 정보, 서버 경로가 노출되지 않는지 확인 | [ ] |

## 3. 상품 등록 및 관리 체크리스트

| 상품 가격 | 가격 범위 검증 | 음수, 문자, 너무 큰 금액이 거부되는지 확인 | [ ] |

## 12. AI 검수 프롬프트

구현 후 다른 AI 또는 코드 리뷰 AI에게 아래 프롬프트로 검수를 요청한다.

```text
이 프로젝트는 UsedCarrot 중고거래 플랫폼이다.
docs/AI_IMPLEMENTATION_SPEC.md와 docs/SECURE_CODING_CHECKLIST.md를 기준으로 구현 상태를 검토해줘.

특히 다음을 집중적으로 확인해줘.
1. 요구사항이 모두 구현되었는지
2. 인증/인가가 서버에서 강제되는지
3. 타인 상품 수정, 타인 채팅 조회, 타인 지갑/CarrotCoin 거래 조회가 불가능한지
4. XSS, SQL Injection, 파일 업로드 공격에 대한 방어가 있는지
5. 비밀번호와 민감정보가 로그에 남지 않는지
6. 신고 누적에 따른 상품 숨김과 사용자 제한/정지가 구현되었는지
7. 테스트 케이스가 충분히 작성되었는지

누락된 항목은 체크리스트 항목 번호와 함께 알려줘.
```
